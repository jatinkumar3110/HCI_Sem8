# Functional Requirements

1. The system shall allow users to add, edit, and delete study tasks.
2. The system shall display tasks in a Kanban board (To Do, In Progress, Done).
3. The system shall allow drag-and-drop of tasks between columns.
4. The system shall allow inline editing of task titles.
5. The system shall provide a modal interface for adding tasks.
6. The system shall display due dates and highlight urgent tasks.
7. The system shall provide a Pomodoro-style study session timer.
8. The system shall show productivity analytics (task completion stats, progress bars).
9. The system shall save all data in local storage.
10. The system shall provide feedback for user actions (success, error, notifications).
