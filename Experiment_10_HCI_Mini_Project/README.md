# Smart Student Study Planner — HCI Mini Project (Experiment 10)

## Overview
A web-based study planner that demonstrates the complete HCI design cycle: user research, requirements, personas, wireframes, prototyping, usability testing, and iteration. The app helps students manage study tasks, track deadlines, and boost productivity.

---

## 1. User Research
- Conducted a survey with 12 students to understand study habits and productivity challenges.
- Key findings: Most use digital tools, struggle with procrastination, want reminders and progress tracking.

## 2. Personas
- Priya Sharma: Forgets deadlines, prefers simple mobile apps.
- Arjun Patel: Procrastinates, likes visual boards, uses laptop.

## 3. Requirements
- User stories and functional requirements defined for all major features (see requirements/).

## 4. Wireframes & Task Flow
- Low-fidelity wireframes for Dashboard, Task Board, Add Task Modal, Timer, Analytics, and Settings.
- Task flow diagram describes navigation between screens (see design/).

## 5. Prototype
- Modern, responsive web app with:
  - Kanban task board (drag & drop)
  - Add/edit/delete/complete tasks
  - Inline editing
  - Modal for adding tasks
  - Deadline tracking
  - Pomodoro timer
  - Analytics/progress view
  - Tabs for navigation
  - Local storage persistence

## 6. Usability Testing
- 3 students completed 5 core tasks.
- Observations: Easy to use, some mobile UI issues, requests for more features.
- See usability_testing/ for details.

## 7. Iteration
- Improved button size, added delete confirmation, highlighted due date field.
- Planned future improvements: sound for timer, recurring tasks.

## Final Reflection
This project demonstrates the full HCI cycle, from research to iteration, resulting in a usable, student-focused productivity tool.
